Phase 1: Context Loading

The session is empty. The naive approach: type your question and hope Claude figures out the codebase. This burns thousands of tokens on exploration that a well-primed session never needs. A well-primed session starts with 10–50k tokens of structured context. A cold session wastes the same budget rediscovering that context through tool calls.

CLAUDE.md and resolvers load project context automatically. Auto-memory (Thariq, 15,845 likes) lets Claude manage its own cross-session memory. HANDOVER.md (Zara Zhang, 3,296 likes) carries decisions, pitfalls, and next steps from the previous session. PAI Statusline v5 (DanielMiessler, 311 likes) shows exactly which custom files loaded.

"Claude now remembers what it learns across sessions — your project context, debugging patterns, preferred approaches — and recalls it later without you having to write anything down."

— Thariq · 15,845 likes · auto-memory launch

Phase 2: Context Management

This is where most practitioners lose without knowing it. The context window is filling with noise — stale tool outputs, failed attempts, verbose error messages. By the time you notice, you're at 80% capacity with 20% signal.

The 1M context paradox: more room doesn't solve the problem — it delays it. At 100k tokens, you hit the wall in 30 minutes. At 1M tokens, you hit it in 3 hours. But the wall is the same wall, and the crash is worse because you've invested more.

Compaction: the invisible infrastructure

Claude Code's autoCompact automatically compresses context as it fills. But the leaked source revealed it has no retry limit — one session had 3,272 consecutive compaction failures before anyone noticed (imyouhu, 6,307 likes).

Context+ (forloopcodes, 2,397 likes) strips noise before it reaches the agent: 6.5k fewer tokens, found the code, fixed it in half the time. RTK (jasonzhou1993, 1,951 likes) reduces token use by up to 60%. The meta-insight: session health is an observability problem. The context window is invisible by default.

Phase 3: Session Handover

The session is ending. What happens next determines whether the work compounds or evaporates.

bcherny's self-improving handover (3,266 likes): "After every correction, end with: 'Update your CLAUDE.md so you don't make that mistake again.' Claude is eerily good at writing rules for itself." The handover isn't just context — it's accumulated judgment.

Session forking (bcherny, 861 likes): fork before the context fills, explore a branch, come back. npx continues (yigitkonur, 2,534 likes) moves context between Claude Code, Gemini, Copilot, and Codex when you hit rate limits. NotebookLM (hooeem, 5,368 likes) makes your HANDOVER.md a persistent retrieval layer outside the context window — 300 sources free, no token cost.

Phase 4: Running Without You

The final evolution: sessions that run when you're not there. Not "ask Claude to do X" but "hire Claude to keep X handled" — a background employee, not a one-shot request.

/loop (bcherny, 12,822 likes): schedule Claude to repeat a task on intervals up to 7 days. /loop 5m /babysit — every 5 minutes, auto-address review comments, auto-rebase, fix builds.

Cloud tasks (noahzweben, 7,469 likes): set a repo, a cron, a prompt. Claude runs on Anthropic infra on schedule. Phone remote (bcherny, 4,615 likes): spawn sessions from the Claude mobile app. --sdk-url (Stan Girard, 4,064 likes): the terminal UI disappears and Claude becomes a WebSocket client — the session interface becomes whatever you want.

What's converging

Three trends are making the session lifecycle as we know it obsolete — by absorbing it into infrastructure.

Trend 1
Compaction becomes infrastructure
The leaked Claude Code compaction engine is already LangChain middleware. Within months, every agent framework will have it. Manual context management becomes optional.
Trend 2
Memory becomes the moat
As sessions get better at persisting knowledge, the accumulated context becomes the competitive advantage — not the model, not the prompts, but the years of decisions your agent remembers.
Trend 3
Sessions become invisible
Between auto-memory, cloud schedules, and remote control, the concept of a "session" dissolves. Claude is always running, always remembering, always available. Managing sessions will feel like managing TCP connections — still there, but abstracted away.
