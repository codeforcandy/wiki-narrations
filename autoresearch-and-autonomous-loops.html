In March 2026, Andrej Karpathy open-sourced AutoResearch — a framework for running autonomous ML experiments overnight. The system is deceptively simple: one file, program.md, defines the experiment template and evaluation criteria. Launch the agent loop. Come back to 100 ranked results. The cost: $25 and a single GPU. Fortune called it "The Karpathy Loop."

The Loop

Everything flows from one file. The program.md defines what to experiment with, how to evaluate, and when to stop. The rest is a thin harness — ~630 lines — that loops, measures, and logs. The intellectual contribution is the meta-prompt. The architecture is the iteration.

The five-step loop animates continuously. The iteration counter in the center ticks up with each cycle.

The critical constraint: AutoResearch requires a numeric score to optimize against. Without one, the loop cannot rank, cannot improve, cannot converge. This is the boundary that defines where the pattern works out of the box — and where it needs adaptation.

The Convergence

AutoResearch is not a tool. It is a pattern. And the pattern converges on three points: always-on optimization via /loop, distributed compute via pooled GPUs, and recursive self-improvement where the loop turns inward. An agent that runs AutoResearch on its own performance — measuring output quality, proposing improvements to its own skills, evaluating, iterating.

Three convergence points: /loop (always-on), distributed compute (SETI@home for AI), self-improvement (recursive agent optimization).

Berkeley's result: told a coding agent to cut its own cost and runtime by 99%. It ran overnight, watching its own logs, editing its own code, rerunning. Result: 98% cost cut, 75% faster. The LEARN phase becomes continuous, not manual.

What's Next

The overnight experiment run becomes a standard part of the SDLC. The coordination layer for distributed compute is the missing piece. The loop that improves things eventually turns inward.
