Three interlocking mechanisms

Compounding happens through three separate systems that reinforce each other. Skills encode judgment, resolvers route context, and a knowledge base grows smarter with every query. Miss any one and the system caps out at "useful tool." Get all three and you're on the exponential curve.

Five principles to encode once

Each one, if you internalize it, removes a class of manual work from your future forever. Read them in the order they're given — the later ones depend on the earlier.

PRINCIPLE 1

Never do one-off work.

If it'll happen again, encode it as a skill. If it should run automatically, put it on a cron. The instruction that resonated with a thousand people: "If I have to ask you for something twice, you failed."

PRINCIPLE 2

CLAUDE.md is a pointer file.

~200 lines of resolver rules. Not a knowledge dump. 20k-line files kill attention; 200 lines pointing to the right docs load context on demand and leave room for real work.

PRINCIPLE 3

Answers go in files, not chat.

Every serious question saved to reports/ and linked from a wiki page compounds into better future answers. Chat disappears; files persist and feed the next compile.

PRINCIPLE 4

Skills auto-match via descriptions.

Claude reads every skill description at startup and matches user intent without you remembering names. Well-written descriptions are the routing layer. Write them for Claude's benefit, not for yours.

PRINCIPLE 5

Health-check your wiki periodically.

LLM knowledge linting catches contradictions, near-duplicates, and missing pages before they accumulate into chaos. The wiki drifts toward higher integrity with zero manual bookkeeping.

Six steps to start compounding this week

In order. Each one independently valuable, together a flywheel. If you can only do one, do step one.

1
Encode your next repeated task as a skill

Whatever you asked Claude to do more than once this week — write a SKILL.md for it tonight. That's the smallest unit of compounding work, and the one most people never do.

2
Audit your CLAUDE.md

If it's over 200 lines, cut it to pointers. Move the content into named documents that resolvers can load on demand. Your attention budget belongs to the work, not the context file.

3
Add a /kb-health skill

Monthly lint pass over wiki/wiki/: contradictions, missing pages for frequently-mentioned concepts, near-duplicates to merge. You already have this one — the narration you're reading is proof it's working.

4
Link every report back to the wiki

After each /kb-report run, add a link from the relevant concept page to the new report. This is the one manual step that makes the graph dense. Without it, reports are islands.

5
Clip aggressively, compile weekly

The capture threshold should be zero friction — use Web Clipper reflexively. Set a weekly reminder (or a cron) to run /kb-compile on whatever accumulated in raw/. Friction is the enemy of capture.

6
Add a wrap-up habit

At the end of significant sessions, have Claude extract what was learned and save it to a session log in raw/. Run /kb-compile to absorb it into the wiki. Sessions stop being ephemeral.
