The Stack: Five Layers

Think of it like a network stack or geological cross-section. Each layer handles a different scale. The bottom is simple and universal; the top is experimental and rare. Most people never get past layer 2. The 10x practitioners live at layers 4 and 5. Click any layer to jump to its detail section below.

1
Flat Files MEMORY.md, CLAUDE.md Markdown the agent reads at session start. Basic continuity. Scale: ~100 files
2
Structured Wiki Obsidian, raw/ → wiki/ → reports/ Compounding knowledge with cross-references. The Karpathy method. Scale: ~1,000 pages
3
Hybrid Retrieval GBrain, pgvector + tsvector + RRF Vector + full-text + ranking. Millisecond retrieval at scale. Scale: 10,000+ docs
4
External Memory NotebookLM, Copilot, Antigravity Memory outside the context window. Token-free persistent retrieval. Scale: unbounded
5
Self-Designed Memory Meta-learning (Jeff Clune, Memento) Agents design their own memory mechanisms. The ultimate moat. Scale: unknown — research frontier

Layer 1 gives you basic continuity — the agent starts primed instead of cold. But at ~100 files it breaks. MEMORY.md becomes "broken by default" — massive, duplicate-heavy, hours to curate.

Layer 2 makes knowledge compound. Article 1 creates 10 pages. Article 50 creates pages that link back to article 1's concepts. At ~500K words, markdown indexes just work. Karpathy himself expected to need fancy RAG and didn't.

Layer 3 is for when wiki alone breaks. GBrain's hybrid retrieval fans out across pgvector HNSW, Postgres tsvector, and Reciprocal Rank Fusion. It ships with an eval harness — P@k, Recall@k, MRR, nDCG@k — while most memory systems have no measurable quality bar.

Layer 4 moves memory outside the context window entirely. NotebookLM bridge: upload your wiki, get 300 free sources, no token cost. The session effectively never ends — it just moves to a cheaper substrate.

Layer 5 is where agents design their own memory. Human-designed schemas are fine for year one. But an agent that discovers its own retrieval patterns — optimized for access you didn't predict — builds a moat that's impossible to replicate by copying the corpus.

RAG vs Wiki: The Decision

The most common question in the memory stack, and the answer is simpler than the debate suggests. Start with the wiki. Add retrieval when you hit the ceiling. Most people over-engineer from day one — building RAG pipelines for 50 documents when grep would have been fine for two years.

The "BS on vector search" position (micLivs, 488 likes): for well-structured vaults, classical text retrieval outperforms vector search. The organization IS the retrieval strategy. Vector search is for when you don't have structure — and the whole point of the memory stack is to create structure.

Memory as Moat

All five layers serve the same strategic purpose: accumulated context becomes a competitive advantage that can't be cloned. The math is brutal in your favor — replication cost grows superlinearly with corpus age.

Replication cost vs. time. By day 365, a competitor cloning your corpus gets a version that's months stale. The cross-links, decision traces, and agent-evolved patterns can't be copied at all.

What's Next: Three Convergence Points

The trajectory is clear. Memory is moving from "something you build" to "something the platform provides." The question is whether you'll be ahead of that curve or catching up to it.

Memory becomes infrastructure

Auto-memory, Copilot memory, Antigravity Knowledge Items
Within a year, every major coding agent ships cross-session memory as default, not add-on. Build your stack now while it's still a differentiator.

Dream cycle becomes standard

Nightly jobs that re-read, extract patterns, rewrite skills
The LangChain co-founder's "dream cycle at 3am" is the natural extension of /loop and /schedule. Expect this as a first-class feature soon.

Self-designed memory wins

Agents that discover their own retrieval patterns
Early research (Jeff Clune, Memento) but directionally the most important. The best memory system is the one the agent designed for itself.
