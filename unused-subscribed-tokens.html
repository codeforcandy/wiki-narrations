The reason people leave subscribed Claude capacity unused is the same reason gym memberships go wasted: the usage model is reactive and manual. You open Claude, ask a question, get an answer, close the tab. If you're not actively prompting, nothing happens. The fix is shifting from a request-response model to a background-worker model.

Each layer covers a different slice of idle time. You need all three to approach your subscription ceiling — hooks for the seams around your active work, loops and schedules for time-driven routines while your machine is on, and cloud schedules for anything that should run when your laptop is closed.

Six steps to stop wasting capacity

In order. Each step is independent and takes under an hour. Start with step one today; by Friday you should have a working loop, a cloud schedule, and a Stop hook notifying you when background jobs finish.

1
Audit your recurring Claude tasks

List everything you've done with Claude in the past two weeks. Mark anything you've done more than once — those are your automation candidates.

2
Set up one /loop today

Pick the highest-frequency recurring task. Most likely: PR review monitoring or a daily standup summary. /loop 5m /babysit. Let it run for a day and see what it handles without you.

3
Set up one cloud-scheduled task this week

Point Claude's scheduled tasks at a repo. Monday morning prompt: "Review open PRs and post a status comment on each." You get the output whether your laptop is on or not.

4
Add claude -w for parallel work

Instead of one Claude finishing task A before starting task B, spin up two worktrees and run both in parallel. Same wall-clock time, double the output.

5
Add a Stop hook that notifies you

When Claude finishes a long background task, you should know.
osascript -e 'display notification "Claude done"' in a Stop hook means you stop checking and start reacting.

6
Run /kb-compile on a schedule

Bookmark sync already runs at 8am via launchd. Add /schedule every sunday run /kb-compile to compile the week's bookmarks into the wiki automatically — zero manual effort, perpetual knowledge growth.
